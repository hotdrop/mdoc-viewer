# XSS検証フィクスチャ

このファイルには、サニタイズ機能が正しく動作するかを確認するための禁止・許可サンプルが含まれます。表示時に危険なリンクやスクリプトが除去されていることを確認してください。

## 禁止されるべき要素

- `[危険なリンク](javascript:alert('XSS!'))`
- `<script>alert('XSS')</script>`
- `<img src="https://example.com/attack.png" alt="should be stripped" />`

上記はいずれもレンダリング時に除去され、プレーンテキストとして扱われるか、表示されないことを想定しています。

## 許可される要素

- `[安全なリンク](../guides/setup-local)`
- ```html
  <code>&lt;a href="/docs/index"&gt;内部リンク&lt;/a&gt;</code>
  ```
- ```bash
  echo "コードブロックはそのまま残る"
  ```

## チェックリスト

1. レンダリング結果に`javascript:`リンクや`img`タグが残っていないこと。
2. `rehype-sanitize`の追加ポリシーで許可した要素が期待通り表示されること。
3. ブロックコード内のHTMLはエスケープされ、ダブルエスケープが起きないこと。

> 万が一禁止要素が表示された場合は、サニタイズポリシーの差分を確認し、`sanitize/policy.extra.ts`の設定漏れがないかチェックしてください。
