# ローカル環境のセットアップ

このドキュメントは`pnpm dev`でアプリを起動し、主要な機能を検証する手順をまとめています。

## 前提条件

- Node.js 20 もしくは 22
- `pnpm install` が済んでいること
- `.env.local` に `LOCAL_DOCS_ROOT=./test_docs/` を設定済みであること

## 起動手順

```bash
pnpm dev
```

起動後は `http://localhost:3000` をブラウザで開き、以下を順番に確認します。

1. トップページのメタデータ（タイトル、説明）が正しく表示される。
2. 認証ガードが未認証状態で`401`を返す。
3. `/viewer` ルートでMarkdownが期待通りに整形される。
4. `/search` ルートで検索件数に応じたフェイルオーバーが起こる。

## ホットリロードの挙動

`test_docs/`内のファイルを保存すると、ビューワが即座に内容を再描画することを確認します。更新のタイミングを把握するため、以下のようにテキストを変更してみてください。

- 任意の段落を追加・削除する
- 見出しレベルを変えて表示スタイルが変わるか確認する
- 相対リンクの参照先を書き換えて、正しく解決されなくなったケースを再現する

## トラブルシューティング

| 症状 | 原因の例 | 対処方法 |
| --- | --- | --- |
| 404が返る | 参照先に`.txt`が存在しない | 対象ファイルがTXT形式か確認する |
| XSS警告が表示された | 危険なリンクが含まれている | [`security/xss-fixtures`](../security/xss-fixtures)で禁止例を確認 |
| レイアウトが壊れる | Markdown構文エラー | Linterかプレビューで確認する |

> **Note:** `pnpm test` を併用してユニットテストが通ることも合わせて確認しておくと安心です。
